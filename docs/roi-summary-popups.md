# 干预收益弹窗 (ROI Summary Popups) 交互与数据说明文档

## 1. 物理干预执行反馈弹窗 (Stretch Execution Feedback Popup)

### 1.1 UI 交互说明
- **触发时机**：用户完成“轻度拉伸”干预（倒计时结束），或手动点击“提前结束”时触发。
- **展现形式**：全屏深色半透明遮罩（`bg-[#0A0A0C]/95` 带模糊效果），居中显示反馈卡片。
- **视觉布局**：
  - **顶部操作区**：悬浮的圆形关闭按钮 `(X)`，点击后关闭弹窗并返回首页。
  - **标题区**：左对齐显示“执行反馈 (EXECUTION FEEDBACK)”，字号较小，颜色偏暗（`text-white/50`）。
  - **核心状态区**：居中显示一个带绿色对勾图标的圆角卡片，确认“拉伸干预已完成”以及“核心躯干能量传输效率已恢复”。
  - **AI 洞察区**：底部居中对齐的说明文本，解释当前干预的实际效果（如：“系统监测到您已持续久坐超过 90 分钟，本次轻度拉伸已有效阻断肌肉僵硬累积...”）。

### 1.2 数据说明
*(注：由于目前缺乏专业的肌电传感器和皮温传感器数据，物理干预暂不提供具体的量化收益数据计算，仅提供定性的执行状态反馈与建议。)*

## 2. 个人效能审计弹窗 (Breathe ROI Popup)

### 2.1 UI 交互说明
- **触发时机**：用户完成“3分钟呼吸法”干预，或手动点击“提前结束”时触发。
- **展现形式**：全屏深色半透明遮罩，居中显示审计卡片。
- **视觉布局**：
  - **顶部操作区**：悬浮的圆形关闭按钮 `(X)`。
  - **标题区**：左对齐显示“个人效能审计 (PERSONAL ROI)”。
  - **核心数据区**：三列等宽的圆角卡片。
    - 左侧卡片：“平均心率降幅”，数值（如 `-12`，绿色），单位 `bpm`。
    - 中间卡片：“算力折算挽回”，数值（如 `+5`，绿色），单位 `%`。
    - 右侧卡片：“执行完赛率”，数值（如 `98`，白色），单位 `%`。
  - **AI 洞察区**：底部包含在一个带边框的深色圆角矩形框内，左对齐显示历史数据对比与干预定性评价。

### 2.2 数据与计算公式
#### 指标 1：平均心率降幅 (Average HR Drop)
- **定义**：呼吸干预期间及结束后，心率相较于干预前的下降幅度，反映交感神经的平复情况。
- **展示逻辑**：负值表示心率下降，使用绿色表示正向收益。
- **计算公式**：
  `平均心率降幅(bpm) = 干预后5分钟平均心率 - 干预前5分钟平均心率`

#### 指标 2：算力折算挽回 (Cognitive Bandwidth Recovery)
- **定义**：将生理指标（心率变异性 HRV 的提升）映射为大脑认知带宽（算力）的恢复比例。
- **展示逻辑**：正值表示算力恢复，使用绿色表示正向收益。
- **计算公式**：
  `算力折算挽回(%) = (干预后HRV - 干预前HRV) / 个人基线HRV * 100% * 认知映射系数`
  *(注：认知映射系数为经验常数，通常在 0.8 - 1.2 之间。)*

#### 指标 3：执行完赛率 (Completion Rate)
- **定义**：用户实际跟随呼吸节奏完成干预的比例。
- **展示逻辑**：百分比显示，使用白色。
- **计算公式**：
  `执行完赛率(%) = (实际参与呼吸干预的时长 / 设定的目标总时长) * 100%`
  *(注：可通过设备监测到的呼吸频率与设定频率的吻合度进行加权调整。)*
